# 画面仕様書案: ユーザー設定・センター専用領域・SS

## 1. 対象画面

本書では、以下の画面を対象とする。

- UserManage: JCLユーザー検索
- UserCreate: JCLユーザー新規登録
- UserEdit: ユーザー設定編集
- ShareArea: センター専用領域
- SS: コラボレーション設定

本書は非プログラマー向けの画面仕様書案であるため、API、Redux、コンポーネント構成などの内部処理は記載対象外とする。

## 2. 共通仕様

管理系のユーザー画面では、画面上部に「ユーザー設定」を表示し、「編集」「登録（AD連携）」のタブで画面を切り替えられる。

操作完了、エラー、確認が必要な操作については、画面上にメッセージまたは確認ダイアログを表示する。

確認ダイアログには「キャンセル」と「OK」を表示する。「キャンセル」を押すと操作を中止し、「OK」を押すと対象操作を実行する。

## 3. UserManage: JCLユーザー検索

### 3.1 画面概要

本画面では、登録済みのJCLユーザーを検索し、対象ユーザーの設定編集画面へ遷移できる。

画面見出しとして「JCLユーザー検索」を表示し、補足として「設定を確認したいユーザーを検索」を表示する。

### 3.2 検索条件

検索条件として、以下の項目を表示する。

| 項目 | 入力方法 | 補足 |
| --- | --- | --- |
| 表示名 | テキスト入力 | 部分一致 |
| ユーザーID | テキスト入力 | 部分一致 |
| メールアドレス | テキスト入力 | `xxxxx.jp`、前方一致 |
| センター | 候補選択 | 複数選択可能 |

### 3.3 操作

「検索」ボタンを押すと、入力した条件に一致するユーザーを検索し、検索結果を表示する。

「クリア」ボタンを押すと、入力中の検索条件を初期化する。

初期表示時は、検索結果一覧および検索結果なしメッセージを表示しない。

### 3.4 検索結果

検索結果がある場合は、「検索結果」見出しと一覧を表示する。

一覧には、以下の列を表示する。

| 列 | 内容 |
| --- | --- |
| ユーザーID | ユーザーを識別するID |
| 表示名 | ユーザーの表示名 |
| メール | ユーザーのメールアドレス |
| BOXアカウント | BOXアカウントが設定されている場合にマークを表示 |
| 操作 | 「選択」ボタンを表示 |

「選択」ボタンを押すと、該当ユーザーのUserEdit画面へ遷移する。

検索結果が複数ページに分かれる場合は、表示中の件数範囲、総件数、ページ操作を表示する。

### 3.5 検索結果なし

検索後、該当するユーザーが存在しない場合は、以下のメッセージを表示する。

> 該当するユーザーが見つかりませんでした。

### 3.6 削除後の表示

UserEdit画面でユーザーを削除して本画面へ戻った場合、以下の完了メッセージを表示する。

> ユーザー{ユーザーID}を削除しました

## 4. UserCreate: JCLユーザー新規登録

### 4.1 画面概要

本画面では、ADユーザーを検索し、JCLユーザーとして新規登録できる。

画面見出しとして「JCLユーザー新規登録」を表示し、補足として「登録したいADユーザーを検索」を表示する。

### 4.2 ADユーザー更新

画面右上に「ADユーザー更新」ボタンを表示する。

初期表示時は、最終更新日時として以下を表示する。

> 最終更新: 未実行

「ADユーザー更新」ボタンを押すと、更新完了メッセージを表示し、最終更新日時を表示する。

完了メッセージ:

> ADユーザーを更新しました

### 4.3 検索条件

検索条件として、以下の項目を表示する。

| 項目 | 入力方法 | 補足 |
| --- | --- | --- |
| 表示名 | テキスト入力 | 部分一致 |
| アカウント名 | テキスト入力 | 部分一致 |
| メールアドレス | テキスト入力 | `xxxxx.jp`、部分一致 |
| 登録状況 | ラジオボタン | 「未登録のみ」「登録済みのみ」「すべて」から選択 |

登録状況の初期値は「すべて」とする。

### 4.4 操作

「検索」ボタンを押すと、入力した条件に一致するADユーザーを検索し、検索結果を表示する。

「クリア」ボタンを押すと、入力中の検索条件を初期化する。

### 4.5 検索結果

検索結果がある場合は、「検索結果」見出しと一覧を表示する。

一覧には、以下の列を表示する。

| 列 | 内容 |
| --- | --- |
| 表示名 | ADユーザーの表示名 |
| アカウント名 | ADユーザーのアカウント名 |
| メール | ADユーザーのメールアドレス |
| 所属 | ADユーザーの所属 |
| 操作 | 登録状態に応じて「登録」または「登録済み」を表示 |

すでにJCLユーザーとして登録済みのユーザーは、操作欄に「登録済み」と表示する。

未登録のユーザーは、操作欄に「登録」ボタンを表示する。

検索結果が複数ページに分かれる場合は、表示中の件数範囲、総件数、ページ操作を表示する。

### 4.6 登録操作

「登録」ボタンを押すと、以下の確認ダイアログを表示する。

タイトル:

> 確認

本文:

> ユーザー{表示名}を登録しますか？

「OK」を押すとJCLユーザーとして登録する。登録に成功した場合は、以下の完了メッセージを表示し、検索結果を再表示する。

> ユーザーを登録しました

補足メッセージとして、登録したユーザーの表示名とアカウント名を表示する。

### 4.7 入力制限

登録時、以下の文字数制限を確認する。

| 項目 | 制限 |
| --- | --- |
| ユーザーID | 31文字以内 |
| 表示名 | 100文字以内 |
| アカウント名 | 100文字以内 |
| メールアドレス | 200文字以内 |

制限を超えている場合は登録せず、以下のエラーメッセージを表示する。

> 入力値が長すぎます

補足として、制限を超えた項目名を表示する。

### 4.8 検索結果なし

検索後、条件に合うユーザーが存在しない場合は、以下のメッセージを表示する。

> 条件に合うユーザーが見つかりませんでした

## 5. UserEdit: ユーザー設定編集

### 5.1 画面概要

本画面では、選択したJCLユーザーの基本情報、所属センター、言語、操作権限を編集できる。

画面見出しとして、以下の形式で表示する。

> ユーザー設定 ＞ {ユーザーID}

補足として以下を表示する。

> ユーザーの基本情報や権限を編集します

### 5.2 画面遷移

「ユーザー検索へ戻る」ボタンを押すと、UserManage画面へ戻る。

対象ユーザー情報の読み込み中は、入力欄の代わりに読み込み中表示を表示する。

### 5.3 ユーザー情報

「ユーザー情報」セクションでは、以下の項目を表示する。

| 項目 | 入力方法 | 備考 |
| --- | --- | --- |
| ユーザーID | 表示のみ | 編集不可 |
| 表示名 | テキスト入力 | 必須 |
| アカウント名（ドメイン） | テキスト入力 | 必須 |
| メールアドレス | テキスト入力 | 必須 |
| 所属センター | 候補選択 | センターを選択 |
| 言語設定 | ラジオボタン | 「日本語」「英語」から選択 |

### 5.4 操作権限

「操作権限」では、権限を選択できる。

権限の選択肢には、権限コードと権限名を表示する。権限を選択すると、選択中の権限内容を表示する。

権限内容には、以下を表示する。

- MyPageデフォルト検索条件1
- MyPageデフォルト検索条件2
- MyPageデフォルト検索条件3
- 各操作権限の有無

各操作権限の有無は、チェックまたは×で表示する。

表示対象の操作権限は以下とする。

| 権限名 |
| --- |
| ジョブ作成 |
| ステータスインポート |
| アクセスユーザー設定 |
| ステータス編集 |
| 保管期限変更 |
| JOB属性編集 |
| 再版 |
| ユーザー特定設定 |
| ログ検索 |
| 管理メニュー |
| NGワード |
| 自動削除フォルダー設定 |

### 5.5 保存

「保存」ボタンを押すと、入力内容を保存する。

保存に成功した場合は、以下のメッセージを表示する。

> 保存しました

保存に失敗した場合は、以下のメッセージを表示する。

> 保存に失敗しました

### 5.6 入力チェック

保存時、以下の内容を確認する。

| 項目 | チェック内容 |
| --- | --- |
| 表示名 | 必須、100文字以内、記号のみ不可 |
| アカウント名 | 必須、100文字以内、1バイト文字のみ、記号のみ不可 |
| メールアドレス | 必須、200文字以内、1バイト文字のみ、記号のみ不可、メールアドレス形式 |
| ユーザーID | 必須、1バイト文字のみ、記号のみ不可 |

入力値に不備がある場合は保存せず、該当するエラーメッセージを表示する。

### 5.7 MyPage検索条件リセット

「MyPage検索条件リセット」セクションでは、対象ユーザーのMyPage検索条件を初期化できる。

説明文として以下を表示する。

> MyPageの検索条件を初期化します。

「リセット」ボタンを押すと、以下の確認ダイアログを表示する。

タイトル:

> 確認

本文:

> ユーザー{ユーザーID}のマイページ検索条件をリセットしますか？

リセットに成功した場合は、以下のメッセージを表示する。

> 検索条件をリセットしました

リセットに失敗した場合は、以下のメッセージを表示する。

> 検索条件のリセットに失敗しました

### 5.8 ユーザー削除

「ユーザー削除」セクションでは、対象ユーザーを削除できる。

説明文として以下を表示する。

> ユーザーを削除します。削除されたユーザーはログインできなくなります。

「削除する」ボタンを押すと、以下の確認ダイアログを表示する。

タイトル:

> 確認

本文:

> ユーザー{ユーザーID}を削除しますか？

「OK」を押すと対象ユーザーを削除する。削除に成功した場合はUserManage画面へ戻り、削除完了メッセージを表示する。

削除に失敗した場合は、以下のメッセージを表示する。

> 削除に失敗しました

## 6. ShareArea: センター専用領域

### 6.1 画面概要

本画面では、利用者がアクセスできるセンター専用領域を一覧表示する。

画面見出しとして「センター専用領域」を表示する。補足として、アクセス可能な共有領域の件数を表示する。

表示例:

> アクセス可能な共有領域 (4件)

### 6.2 検索

画面右上に検索欄を表示する。

検索欄のプレースホルダーは以下とする。

> 領域を検索...

検索欄に文字を入力すると、フォルダ名または領域名に一致する領域を絞り込む。

### 6.3 一覧表示

一覧には、利用者がアクセス可能な領域を表示する。自部署の領域は一覧の上部に表示する。

各領域には、以下を表示する。

| 項目 | 内容 |
| --- | --- |
| フォルダ名 | Box上のフォルダ名 |
| 領域名 | センターまたは用途の名称 |
| ゲストラベル | ゲスト領域の場合に表示 |
| Box ブラウザ | BoxのWeb画面を開くボタン |
| Box Drive | Box Driveで開くためのボタン |

現状の表示対象は以下とする。

| フォルダ名 | 領域名 | ゲスト |
| --- | --- | --- |
| qms | QMS事務局 | なし |
| ems | EMS事務局 | なし |
| tg-room | 統合サーバー | あり |
| JCLGD1SWDV | 第1ソフトウェア開発センター | なし |

### 6.4 領域操作

「Box ブラウザ」ボタンを押すと、該当領域をBoxのWeb画面で開く。

「Box Drive」ボタンを押すと、該当領域をBox Driveで開くためのリンクを開く。

自部署の領域には「コラボレーション設定」ボタンを表示する。押すと、該当領域のSS画面へ遷移する。

管理者向けに「センターメンバー一覧」ボタンを表示する。現状では、遷移先または具体的な動作は未定義とする。

### 6.5 検索結果なし

検索条件に一致する領域がない場合は、以下のメッセージを表示する。

> 該当する領域が見つかりません

## 7. SS: コラボレーション設定

### 7.1 画面概要

本画面では、センター専用領域内のフォルダを確認し、フォルダごとのコラボレーターを管理できる。

ShareArea画面から選択された領域を対象に表示する。許可されていないフォルダIDで直接アクセスした場合は、ShareArea画面へ戻す。

### 7.2 ヘッダー・戻る操作

画面上部に、対象領域名を表示する。

「センター専用領域に戻る」ボタンを押すと、ShareArea画面へ戻る。

Box接続中は、以下のメッセージを表示する。

> Box に接続中...

### 7.3 フォルダ操作

フォルダ操作エリアには、以下を表示する。

| 操作 | 内容 |
| --- | --- |
| 戻る | 直前に表示していたフォルダへ戻る |
| 進む | 戻る操作後、次のフォルダへ進む |
| パス表示 | 現在表示中フォルダのパスを表示 |
| パスをコピー | 現在のフォルダパスをコピー |
| Web Boxで開く | 現在のフォルダをBoxのWeb画面で開く |
| Box Driveで開く | 現在のフォルダをBox Driveで開く |

パスコピーに成功した場合は、以下のメッセージを表示する。

> パスをコピーしました

パスコピーに失敗した場合は、以下のメッセージを表示する。

> コピーに失敗しました

Box Driveで開く操作をした場合は、以下のメッセージを表示する。

> エクスプローラーで開きます

### 7.4 フォルダ表示

画面左側にBoxのフォルダ内容を表示する。

利用者はフォルダを移動できる。フォルダを移動すると、画面右側のコラボレーター一覧も現在のフォルダに合わせて更新する。

深い階層のフォルダを復元中の場合は、読み込み中表示を表示する。

### 7.5 コラボレーター追加

画面右側に、現在のフォルダ名と「コラボレーターを追加」欄を表示する。

追加欄では、以下を指定できる。

| 項目 | 入力方法 | 備考 |
| --- | --- | --- |
| 部署・社員 | 候補検索 | プレースホルダーは「部署・社員を検索...」 |
| ロール | 選択 | 「Editor」「Viewer」から選択 |

ロールの初期値は「Viewer」とする。

部署または社員が選択されるまで、「追加」ボタンは押せない。

「追加」ボタンを押すと、以下の確認ダイアログを表示する。

タイトル:

> コラボレーションを追加しますか？

本文:

> {対象者名} を {ロール} で追加します。

「OK」を押すとコラボレーターを追加する。追加に成功した場合は、以下のメッセージを表示し、一覧を最新状態に更新する。

> {対象者名} を追加しました

同じコラボレーターがすでに設定されている場合は、以下のメッセージを表示する。

> 同じコラボレーターは既に設定されています

追加に失敗した場合は、以下のメッセージを表示する。

> コラボレーターの追加に失敗しました

追加自体は完了したが一覧更新に失敗した場合は、以下のメッセージを表示する。

> 追加は完了しましたが一覧の更新に失敗しました

### 7.6 コラボレーター一覧

「コラボレータ一覧」には、現在のフォルダに設定されているコラボレーターを表示する。

一覧の読み込み中は、読み込み中表示を表示する。

コラボレーターが設定されていない場合は、以下のメッセージを表示する。

> コラボレーターは設定されていません

各行には、以下を表示する。

| 項目 | 内容 |
| --- | --- |
| 種別 | ユーザーまたは部署を示すアイコン |
| 対象者名 | コラボレーター名 |
| ロール | EditorまたはViewer |
| 継承元パス | 親フォルダから継承された設定の場合に表示 |
| 削除操作 | 編集可能な設定の場合に表示 |

親フォルダから継承された設定は、継承元のパスを表示する。

編集できない設定は読み取り専用として表示し、ロール変更や削除はできない。

### 7.7 ロール変更

編集可能なコラボレーターは、ロールを変更できる。

ロールを変更すると、確認ダイアログを表示する。

通常の設定の場合、本文は以下の形式で表示する。

> {対象者名} のロールを {ロール} に変更します。

継承された設定の場合、本文は以下の形式で表示する。

> {対象者名} の継承元コラボレーション（{継承元パス}）を {ロール} に変更します。

変更に成功した場合は、以下のメッセージを表示し、一覧を最新状態に更新する。

> {対象者名} のロールを更新しました

ロール変更に失敗した場合は、以下のメッセージを表示する。

> コラボレーターのロール更新に失敗しました

更新自体は完了したが一覧更新に失敗した場合は、以下のメッセージを表示する。

> 更新は完了しましたが一覧の更新に失敗しました

### 7.8 コラボレーター削除

編集可能なコラボレーターは削除できる。

削除ボタンを押すと、確認ダイアログを表示する。

通常の設定の場合、本文は以下の形式で表示する。

> {対象者名} のコラボレーションを削除します。

継承された設定の場合、本文は以下の形式で表示する。

> {対象者名} の継承元コラボレーション（{継承元パス}）を削除します。

削除に成功した場合は、以下のメッセージを表示し、一覧を最新状態に更新する。

> {対象者名} を削除しました

削除に失敗した場合は、以下のメッセージを表示する。

> コラボレーターの削除に失敗しました

削除自体は完了したが一覧更新に失敗した場合は、以下のメッセージを表示する。

> 削除は完了しましたが一覧の更新に失敗しました

### 7.9 一覧取得エラー

コラボレーター一覧の取得に失敗した場合は、以下のメッセージを表示する。

> コラボレーター一覧の取得に失敗しました

## 8. 補足・確認事項

以下は、仕様確定時に確認が必要な事項とする。

- ShareAreaの「センターメンバー一覧」ボタンの遷移先または操作内容
- ShareAreaで自部署として扱うセンターの判定方法
- ShareAreaに表示する領域の最終的な一覧
- UserCreateの「ADユーザー更新」ボタンで実際に更新する対象と更新タイミング
