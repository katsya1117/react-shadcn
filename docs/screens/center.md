# センター管理 画面仕様書

センターの検索・作成・編集を行う画面群。タブで3画面が切り替わる。

---

## 画面構成

```
CenterTabsShell（タブ共通ラッパー）
├── [センター検索/編集] タブ → CenterManage
├── [センター編集]     タブ → CenterEdit（センター選択後に遷移）
└── [新規作成]         タブ → CenterCreate
```

ルーティング：

| パス | コンポーネント |
|------|---------------|
| `/center/manage` | CenterManage |
| `/center/edit/:center_cd` | CenterEdit |
| `/center/create` | CenterCreate |

---

## CenterManage — センター検索 / 一覧

**ファイル：** `src/pages/CenterManage.tsx`

### 概要

センター名またはメンバー名で検索し、結果一覧からセンターを選んで編集画面へ遷移する。

### 検索フォーム

| 項目 | 種別 | 動作 |
|------|------|------|
| センター名 | テキスト入力 | センター名・拠点・メンバー数に対してフィルタリング |
| メンバー | テキスト入力 | ユーザー名での逆引き検索（現在はUI上のみ、フィルタ未接続） |
| 検索ボタン | ボタン | 押下後に検索結果テーブルを表示する |

「検索」ボタンを押す前はテーブルを表示しない（「検索ボタンを押してください」と表示）。

### 検索結果テーブル

| カラム | 内容 |
|--------|------|
| センター名 | センターの名称 |
| メンバー数 | 所属するメンバーの人数 |
| タイプ | `所属` / `カスタマイズ` |
| 拠点 | センターの所在拠点 |
| 操作 | 「選択」ボタン → CenterEdit へ遷移 |

ヒットなしの場合は「該当するセンターが見つかりませんでした。」を表示。

### 遷移

- 「選択」押下 → `/center/edit/:center_cd` へ遷移

---

## CenterEdit — センター編集

**ファイル：** `src/pages/CenterEdit.tsx`

### 概要

選択したセンターの詳細情報を確認・編集する。基本設定・詳細情報・メンバー管理・センター削除の4セクション構成。

### ヘッダー

- 「戻る」ボタン：ブラウザ履歴を1つ戻る
- センターコード・拠点をバッジで表示
- 「センター一覧へ戻る」ボタン：CenterManage へ遷移

### セクション1：基本設定

| 項目 | 種別 | 備考 |
|------|------|------|
| センターコード | テキスト（読み取り専用） | URLパラメータから取得 |
| センター名 | テキスト入力 | |
| タイプ | テキスト入力 | |
| 拠点 | テキスト入力 | |

「保存」ボタンで送信（現在はUI上のみ）。

### セクション2：詳細情報（容量・比率）

| 項目 | 備考 |
|------|------|
| 割り当て容量 (Kb) | ヒント: TB換算値を補足表示 |
| 全体使用量 (Kb) | ヒント: Mb換算値を補足表示 |
| 共有領域使用量 (Kb) | |
| JOB比率 (%) | |
| 使用率 (%) | |

「保存」ボタンで送信（現在はUI上のみ）。

### セクション3：メンバー

現在のメンバー一覧を表示し、ゲストの追加・削除・管理者権限の変更ができる。

**テーブルカラム：**

| カラム | 内容 |
|--------|------|
| ユーザーID | 内部ID（`u001` など） |
| ユーザー名 | 氏名。ゲストには「ゲスト」バッジを表示 |
| 権限 | `標準` / `閲覧のみ` |
| 管理者 | チェックボックスで切り替え可能 |
| 操作 | ゲストのみ「削除」ボタンを表示 |

**メンバー種別：**

| 種別 | 説明 |
|------|------|
| 所属 | センターに正式に所属するメンバー。削除ボタンなし |
| ゲスト | 招待されたメンバー。削除可能 |

**管理者表示：**
カードヘッダーに「管理者：〇〇、△△」と現在の管理者名を一覧表示。

**ゲスト追加（Sheetパネル）：**

右からスライドインするSheetで入力。

| 項目 | 必須 |
|------|------|
| ユーザーID | ○ |
| 表示名 | ○ |

追加後は権限が「閲覧のみ」・管理者フラグ OFF で登録される。
「追加」ボタンはユーザーID・表示名の両方が入力済みの場合のみ有効。

**削除：**
確認ダイアログ（ConfirmButton）を経て削除。削除後はトーストで通知。

### セクション4：センター削除

赤いタイトル（`text-destructive`）で危険操作であることを強調。
「削除する」ボタン押下 → 確認ダイアログ → 削除実行 → CenterManage へ遷移。

## CenterCreate — センター新規作成

**ファイル：** `src/pages/CenterCreate.tsx`

### 概要

新規センターを作成する画面。現在はモックのプレースホルダーのみで、入力フォームは未実装。

### 現在の状態

「入力項目を追加して作成処理を実装してください。」というメッセージのみ表示。

### 実装予定

CenterEdit の基本設定セクションを参考に、以下の項目を実装する想定：

- センターコード（手動入力 or 自動採番）
- センター名
- タイプ（`所属` / `カスタマイズ`）
- 拠点
- 初期メンバーの設定

## 共通：CenterTabsShell

**ファイル：** `src/pages/CenterTabsShell.tsx`

3画面を包むタブナビゲーションラッパー。`active` プロパティで現在のタブを指定する。

| `active` 値 | 表示タブ |
|------------|---------|
| `"edit"` | センター検索/編集 |
| `"new"` | 新規作成 |

## データモデル（現在はモック）

```ts
type Center = {
  code: string;            // センターコード（例: "c001"）
  name: string;            // センター名
  members: number;         // メンバー数
  type: "所属" | "カスタマイズ";
  site: string;            // 拠点名
};

type Member = {
  id: string;
  name: string;
  role: string;            // "標準" | "閲覧のみ"
  isAdmin: boolean;
  guestLabel: "所属" | "ゲスト";
};
```

CenterManage と CenterEdit はそれぞれローカルにモックデータを持っており、
実装時は API・Redux に置き換える。
